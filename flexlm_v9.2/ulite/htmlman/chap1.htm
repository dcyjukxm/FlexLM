Chapter 1

Introduction

1.1 Overview of FLEXlm Ultralite

FLEXlm Ultralite is a software licensing library for devices with the following characteristics:

  • Limited amount of random access memory (RAM)
  • Limited development environment
  • Limited or no network connectivity

FLEXlm Ultralite can be ported to machines and devices that do not have these limitations, but machines and devices without these limitations are better supported by FLEXlm.

To accommodate these limited devices, the FLEXlm Ultralite library has the following characteristics:

  • Small footprint
  • No dynamic memory allocation
  • No global or static variables
  • No floating point variables or intermediate floating point calculations
  • No calls to external functions
  • Uncounted, unexpiring, unversioned license models only

1.2 About This Manual

This manual, the FLEXlm Ultralite Programmers Guide, includes a complete description of the application programming interface (API) that is used to incorporate FLEXlm Ultralite license management into your application.

1.3 Related Documentation

To understand FLEXlm Ultralite, you should understand the basic workings of FLEXlm. Refer to the following documents for basic FLEXlm issues:

  • The FLEXlm Programmers Guide
    Provides an introduction to FLEXlm, instructions for evaluating FLEXlm on UNIX and Windows systems, descriptions of the Trivial and Simple APIs, and guidelines for integration of FLEXlm into your application.
  • The FLEXlm Reference Manual
    Provides a comprehensive description of all other aspects of FLEXlm from the software developer's perspective, including a complete description of the FLEXible API, the most complete API available for license management.
  • The FLEXlm Ultralite Release Notes
    Provides platform-specific information for this version of FLEXlm Ultralite. This information includes the platform build environment specifics, contents of the kit, memory usage issues and an API coding example.

1.4 Typographic Conventions

The following typographic conventions are used in this manual:

  • The first time a new term is used it is presented in italics.
  • Commands and path, file, and environment variable names are presented in a fixed_font.
  • Other variable names are in an italic_fixed_font.
  • API function calls are in a sans-serif font.
